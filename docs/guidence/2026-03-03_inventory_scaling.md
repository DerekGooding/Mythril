# Guidance Request: Inventory Scaling & Stack Limits

**Date:** 2026-03-03
**Agent:** Gemini CLI

## Context
Currently, the inventory allows infinite stacks of materials, while magic has a "Magic Capacity" (increased by Magic Pocket abilities). As the number of items grows, the vertical list in the UI may become unwieldy.

Proposed Options:
1.  **Category Filtering**: Add UI filters (Materials, Spells, Quest Items) to the inventory panel.
2.  **Physical Storage**: Introduce a "Warehouse" or "Chest" system where users must move excessive items to store them, keeping the "active" inventory small.
3.  **Stack Caps**: Implement hard caps on item stacks, requiring players to refine or spend them before they can earn more.

## Questions
1. Should non-magic items (Materials/Gold) have capacity limits similar to Magic?
2. Do you prefer UI-only solutions (filtering) or gameplay-mechanic solutions (storage management)?
